# 🤖 Medicare FAQ Chatbot — Retrieval-Based NLP with Text Embeddings

A Python chatbot that answers Medicare-related questions using **semantic text embeddings** and **cosine similarity**. Rather than relying on keyword matching, it understands the *meaning* of a question and retrieves the most relevant FAQ from its knowledge base.

---

```

├── chatbot.py              # Builds & caches the FAQ knowledge base as embeddings
├── embedding.py            # Loads embeddings and matches user queries at runtime
```

## 🧠 How It Works

This project demonstrates a classic **retrieval-based QA pipeline**:

```
  ┌──────────────────────────────────────────────────────┐
  │  1. FAQ questions → SentenceTransformer → vectors    │
  │  2. Save vectors to embeddings.csv                   │
  │  3. User query   → SentenceTransformer → query vec   │
  │  4. cosine_similarity(query_vec, all_faq_vecs)       │
  │  5. Return the FAQ with the highest similarity score  │
  └──────────────────────────────────────────────────────┘
```

### Why Embeddings Instead of Keywords?

| Keyword Matching | Embedding-Based Matching |
|-----------------|-------------------------|
| "Medicare card" only matches "Medicare card" | "replacement ID" can match "replacement Medicare card" |
| Fails on synonyms | Understands synonyms and paraphrases |
| Case/spelling sensitive | Semantic understanding |
| Fast but rigid | Fast and flexible |

## 🚀 Quick Start

### 1. Install Dependencies

```bash
pip install sentence-transformers pandas scikit-learn numpy
```

### 2. Build the Knowledge Base

Run `chatbot.py` to encode FAQ questions and generate `embeddings.csv`:

```bash
python chatbot.py
```

### 3. Query the Chatbot

Run `embedding.py` to match a user query against the knowledge base:

```bash
python embedding.py
```

**Example output:**

```
Query: How can Medicare help me?
Most similar FAQ: What is Medicare and who can get it?
Similarity score: 0.7842
```

### 4. 📓 Follow the Interactive Tutorial

For a full guided walkthrough with explanations and visualizations, open:

```bash
jupyter notebook chatbot_tutorial.ipynb
```

## 🔬 The Model: `all-MiniLM-L6-v2`

This project uses the `all-MiniLM-L6-v2` model from the Sentence Transformers library:

- **Embedding size:** 384 dimensions
- **Speed:** Very fast (suitable for real-time inference)
- **Trained on:** Large-scale semantic similarity datasets
- **Best for:** Sentence-level semantic similarity tasks

The model converts any sentence into a 384-dimensional vector where **semantically similar sentences produce similar vectors**.

---

## 📐 Cosine Similarity Explained

Cosine similarity measures the angle between two vectors, returning a score between 0 and 1:

```
similarity = 1.0  →  Identical meaning
similarity = 0.7  →  Closely related
similarity = 0.4  →  Loosely related
similarity = 0.0  →  Unrelated
```

## 💡 Extending This Project

- **Add answers**: Store answer text alongside questions and return actual answers
- **Scale the knowledge base**: Add hundreds of FAQ pairs from real documentation
- **Vector database**: Replace CSV with [FAISS](https://github.com/facebookresearch/faiss), [Chroma](https://www.trychroma.com/), or [Pinecone](https://www.pinecone.io/)
- **Web interface**: Wrap in [Streamlit](https://streamlit.io/) or Flask for a browser UI
- **Confidence threshold**: Return "I don't know" when similarity is too low
- **Hybrid retrieval**: Combine semantic search with BM25 keyword search
